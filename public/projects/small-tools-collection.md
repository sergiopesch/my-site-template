## Keep the scope small

Each utility should do one job clearly. Describe the repeated task, the smallest useful intervention, and how you know the tool improved the workflow.

When the code cannot be shared, link to a [public demonstration](https://example.com/projects/small-tools-collection) or a [related write-up](https://example.com/writing/small-useful-tools) instead.

## Protect what is private

Private repository metadata is deliberately excluded from public URLs, page metadata, feeds, and discovery outputs. The test suite verifies that boundary on every build.
