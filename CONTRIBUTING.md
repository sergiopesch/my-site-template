# Contributing

Thanks for improving the template.

1. Keep changes generic. Do not add personal domains, analytics IDs, secrets, or content that generated sites would inherit accidentally.
2. Preserve one normalized content model for HTML and every machine-readable representation.
3. Add or update regression tests for behavior changes.
4. Run `npm run check` and `npm run test:e2e` before opening a pull request.
5. Regenerate public artifacts with `npm run generate:machine`; never edit generated files by hand.

Pull requests should explain the user-facing outcome, compatibility implications, performance impact, security/privacy impact, and the checks performed.
